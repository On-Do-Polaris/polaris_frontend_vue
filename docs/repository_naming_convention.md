# Repository Naming Convention

polaris 프로젝트의 Repository Naming Convention 정의

## 1. 목적

- 레포지토리 이름을 통한 역할과 기술 스택의 명확한 파악
- 통일된 규칙 적용으로 협업 효율성 향상

## 2. 기본 작성 규칙

```
<polaris>-<카테고리>-<스택>
```

## 3. 세부 규칙

### 3.1. 필수 준수사항

- 전체 소문자만 사용
- 하이픈(-) 으로 구분
- 3 단어 이내 구성
- 역할이 명확히 드러나는 단어 사용
- 언더스코어(\_) / 공백 / 대문자 사용 금지
- 기술 버전(예: python311, vue3 등)은 레포 이름에 포함 금지
- 버전은 README 또는 환경 파일에서 관리

## 4. 카테고리 정의

레포지토리 목적을 명확하게 하기 위해 아래 카테고리를 사용

| 카테고리   | 설명                    |
| ---------- | ----------------------- |
| `frontend` | 프론트엔드 애플리케이션 |
| `backend`  | 백엔드 서버 및 Agent    |
| `infra`    | 인프라·배포·IaC         |

## 5. 작성 예시

### 5.1. 프론트엔드

```
polaris-frontend-vue
```

### 5.2. 백엔드

```
polaris-backend-java
polaris-backend-python
```

### 5.3. 인프라

```
polaris-infra-sh
```

## 4. 검토 체크리스트

- [ ] 기본 작성 규칙을 지켰는가
- [ ] 모든 문자가 소문자인가
- [ ] 하이픈으로만 구분했는가
- [ ] 정해진 카테고리에서 사용하였는가
- [ ] 3 단어 이내인가
- [ ] 기술 버전이 이름에 포함되지는 않았는가
- [ ] 불필요하게 추상적이거나 테스트용 이름이 아닌가
